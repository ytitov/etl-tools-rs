This library provides a foundation to construct pipelines using several basic principles. There are stream producers and consumers, which must implement the traits crate::datastore::DataSource and crate::datastore::DataOutput. etl_job::job::JobRunner is used to construct pipelines using these elements. It also provides some simple state management which is loaded and saved using the crate::datastore::simple::SimpleStore trait which is designed for loading whole files It is important to note that the purpose of this library is to provide a framework to manage and organize pipelines. It is not meant to replace data anylitical tools or databases. The main use case for the author is performing 100 G JSON data merges and mapping them to a schema located on an MySql database.
